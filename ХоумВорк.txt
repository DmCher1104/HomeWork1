1. Скачать и установить JDK
2. Настроить переменные среды (или создать, если таковых нет) PATH, CLASSPATH, JAVA_HOME
2.1 Проверить установку джавы при помощи команды java -version
3. Как правило Java код мы храним в папке src, который надо преобразовать в байт код при помощи  java компилятора (!!! Если будем находиться непосредственно в папке с исходниками (С:\Aproject\src\org\run), то компилятор будет искать классы org.run.Application.java и остальные из папки С:\Aproject\src\org\run\org\run\Application.java и остальные)
	"C:\Program Files\Java\jdk1.8.0_211\bin\javac.exe" org\run\Application.java -d .. \target\
4. Наши сгенерированные файлы, как правило размещаются в папке target ( мы их туда поместили при помощи команды -d .. \target\ ), которые можем запустить при помощи JVM.
	"C:\Program Files\Java\jdk1.8.0_211\bin\java.exe" org.run.Application 
4.1 Чтобы не прописывать полные пути к выполняемым файлам
	"C:\Program Files\Java\jdk1.8.0_211\bin\javaс.exe"
	"C:\Program Files\Java\jdk1.8.0_211\bin\java.exe"
необходимо настроить системные переменные среды PATH, CLASSPATH и JAVA_HOME, для   этого и был предложен пункт 2. Итог компиляции и запуска в упрощенном виде 	представлен ниже:
	javac org\run\Application.java
	java org.run.Application
5. Classpath – это набор классов, доступных для нашей JVM, работающей в текущий 
момент. Мы используем переменную classpath, чтобы сообщить JVM, где искать файлы .class в 
файловой системе.
6. Конечный файл обычно представляет собой наше приложение, либо наша библиотека… В общем то, что мы разрабатывали. Обычно все упаковывается в 1 файл .jar (Java archive) или .war (web archive)
Чтобы упаковать в архив  необходимо быть в папке \target (C:\Aproject\target) воспользуемся командой jar.
   	jar cvf MyProgram.jar .    
	“.” – означает, что в архив будут добавлены все файлы из текущей папки, где мы находимся.
7.  Это наша программа либо какая-нибудь библиотека, которую мы разрабатывали.Чтобы запустить наше приложение .jar необходимо, чтобы в манифесте была указана точка входа main(). 
	Main-Class: org.run.Application
